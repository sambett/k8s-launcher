! Read before you start - Platform requirements
Supported OS Ubuntu 22.04 LTS or Ubuntu 24.04 LTS - all nodes (control plane + workers) must run the same version.
Architecture amd64 (x86-64) only.
Edition Use Ubuntu Server images on all nodes for a cleaner, more predictable setup.
Why same OS? Longhorn relies on kernel modules - iSCSI, device-mapper - that differ between Ubuntu LTS generations. A mixed-OS cluster produces subtle, hard-to-diagnose storage failures under load.

Phase 0 - Install Ansible on the controller

The controller is the machine that runs Ansible and orchestrates the cluster. It must NOT be one of your cluster nodes - it stays separate. If you are running this launcher directly on the controller, tick the checkbox below.

Hostname best practice
The hostname entered here and the machine's real hostname must be the same for best practices. This helps Ansible and Kubernetes refer to the machine consistently later. Use lowercase letters, numbers, and hyphens only, because Kubernetes hostnames follow lowercase DNS-style naming rules.

Phase 2 - Preflight checks

This step checks whether all selected machines are ready for installation. It verifies operating system, memory, free disk space, swap, Python, SSH bootstrap, and whether the machine is still in a clean state.
If you continue after failed checks, deployment may fail or behave unexpectedly. It is safer to correct the reported issues first.
