⚠️ Read before you start — Platform requirements
Supported OS Ubuntu 22.04 LTS or Ubuntu 24.04 LTS — all nodes (control plane + workers) must run the same version.
Architecture amd64 (x86-64) only.
Why same OS? Longhorn relies on kernel modules — iSCSI, device-mapper — that differ between Ubuntu LTS generations. A mixed-OS cluster produces subtle, hard-to-diagnose storage failures under load.

Phase 0 — Install Ansible on the controller

The controller is the machine that runs Ansible and orchestrates the cluster. It must NOT be one of your cluster nodes — it stays separate. If you are running this launcher directly on the controller, tick the checkbox below.

Phase 1 — SSH key bootstrap (cluster nodes)

Fill in the IP, hostname, and SSH user for each cluster node. The password is used once to push the SSH key — it is never saved anywhere.

Phase 2 — Preflight checks

Verifies OS, RAM, disk, swap, Python 3, and clean state on every node.
