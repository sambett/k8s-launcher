Phase 0 — Install Ansible on the controller

The controller is the machine that runs Ansible and orchestrates the cluster. It must NOT be one of your cluster nodes — it stays separate. If you are running this launcher directly on the controller, tick the checkbox below. Otherwise enter the controller's IP, SSH user, and password.

Phase 1 — SSH key bootstrap (cluster nodes)

Fill in the IP, hostname, and SSH user for each cluster node. The password is used once to push the SSH key — it is never saved anywhere.

Phase 2 — Preflight checks

Verifies OS, RAM, disk, swap, Python 3, and clean state on every node.
