GitLab settings

🌱 Seed — first group and test user (created automatically after deploy)

After GitLab is deployed, Ansible will automatically create one group and one test user and add both to the group. This means JupyterHub login works immediately — no manual steps needed.

Group name — the GitLab group users must belong to in order to access JupyterHub. You can create more groups later via the Admin Dashboard.

Test username / password — a real GitLab account you can use to log into JupyterHub and verify the full flow end to end. Change the password to something strong before going to production.
⚠ GitLab credentials already exist. Re-deploying will rotate the admin token. Use the Reset tab → Reset GitLab to start fresh, or tick below to force.

Step 3 — Deploy GitLab

Installs GitLab CE, configures the registry, creates all tokens and the OAuth app, then seeds the initial group and test user automatically.
