What is this token and why does the dashboard need it?

The dashboard talks to your GitLab instance (loading…) on your behalf. Every action — listing groups, creating users, browsing the registry, importing images — is an authenticated GitLab API call. The token is the credential that authorises those calls.

How to get a new token
  1. Open your GitLab instance → loading…
  2. Log in as an admin account
  3. Click your avatar (top-right) → Edit profile
  4. Left sidebar → Access Tokens
  5. Click Add new token
  6. Fill in:
    • Name: workbench-admin
    • Expiry: 1 year recommended
    • Scopes: check api and read_registry
  7. Click Create personal access token
  8. Copy the token — starts with glpat- and shown only once
  9. Paste it below and click Save & Verify
⚠ The token is stored in loading… on the control plane node — readable only by root and the service user. It is never committed to git or exposed in API responses.
